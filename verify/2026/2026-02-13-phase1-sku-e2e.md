# 2026-02-13 --- Phase 1 SKU End-to-End Verification

## Purpose

This document records the manual verification procedure for Phase 1 SKU
vertical slice.

The goal is to confirm correct integration across database, backend, and
frontend layers.

------------------------------------------------------------------------

## Environment

-   Docker Compose (PostgreSQL, Redis)
-   FastAPI backend
-   React + Vite frontend
-   Ant Design UI framework

------------------------------------------------------------------------

## Verification Steps

### 1. Infrastructure

Start and verify Docker services:

``` bash
docker compose -f src/env/docker/docker-compose.yml ps
```

Confirm PostgreSQL and Redis are running.

------------------------------------------------------------------------

### 2. Backend

Start FastAPI service:

``` bash
cd src/dev/backend
.venv/bin/uvicorn app.main:app --reload --port 8000
```

Verify health endpoint:

``` bash
curl http://localhost:8000/health
```

Verify SKU endpoint:

``` bash
curl http://localhost:8000/skus
```

Expected response:

``` json
[]
```

------------------------------------------------------------------------

### 3. Frontend

Start frontend dev server:

``` bash
cd src/dev/frontend
pnpm dev
```

Open browser:

    http://localhost:5173

Navigate to `/skus` and confirm empty table is displayed.

------------------------------------------------------------------------

### 4. Data Validation (Optional)

Insert sample data:

``` bash
docker exec -i docker-postgres-1 psql -U wms wms_db -c \
  "INSERT INTO skus (code, name, unit) VALUES ('SKU-001', 'Widget A', 'pcs'), ('SKU-002', 'Widget B', 'box');"
```

Refresh `/skus` page and verify rows appear.

------------------------------------------------------------------------

## Result

All verification steps completed successfully.

Phase 1 SKU vertical slice was validated end-to-end.

------------------------------------------------------------------------

## Notes

-   Container names may vary across environments.
-   Use `docker compose ps` to locate correct service name if needed.
-   These steps should be repeated after major backend or frontend
    changes.
